## Features

1. **shelter-pair (d4d6b9a)** — `Storm shelter co-op`. Two shelters
   within Chebyshev distance 2 of each other widen each one's effective
   coverage from a 3x3 (SHELTER_RADIUS=1) to a 5x5
   (SHELTER_RADIUS_PAIRED=2) for the next storm. Both are still
   consumed when the storm hits.
   - `effectiveRadius(world, shelter)` is the per-shelter accessor;
     `isPaired(world, shelter)` is the boolean variant.
   - `consumeShelteringShelters` snapshots radii BEFORE mutating the
     shelters[] array so removing one half of a pair never shrinks the
     surviving partner's coverage mid-loop.
   - `drawShelterSprite` now takes an optional `paired` flag — paired
     shelters render a two-puff cloud sigil so the pair reads visually.
   - Game.ts placement toast now names the pair when the new shelter
     lands within range of an existing one.
   - 11 new tests in `tests/storm-shelter-pair.test.ts`.

2. **rare-fertilizer (4aac877)** — `Fertilizer-of-the-week`. Per-season
   "rare day" derived from a deterministic 32-bit hash of season+1
   (spring=3, summer=6, fall=5, winter=2). Compost batches whose
   finishOnDay matches the rare day mint into
   RARE_FERTILIZER_INVENTORY_KEY worth +4 streak instead of +2.
   - `compostTick(world, player, today, season=0)` got the season
     param; old callers still work but lose the rare distinction.
     game.ts now passes `time.season`.
   - `applyFertilizer` prefers rare bags first when both are in the bag;
     ApplyOutcome carries `bonus` + `rare` so the toast can split the
     wording.
   - The 7-key apply toast names "Rare fertilizer applied" + the actual
     bonus.
   - Dawn compost toast names the rare delta when any rare bags drop.
   - 9 new tests in `tests/compost-rare.test.ts`. Existing compost test
     suite re-pinned to season=1 so the fixture finishOnDay (3) stays
     non-rare.

3. **heritage-chicks (d1f9812)** — `Hatchery rare-chicken roll`. 18% of
   hatched chicks are now HERITAGE breeds (HERITAGE_HATCH_RATE).
   Heritage chickens get HERITAGE_FANCY_BONUS=0.15 added to their
   per-chicken fancy-egg rate, so a basic-tier 0-happiness heritage
   chick lays fancy eggs ~23% of the time vs the regular 8%.
   - `rollHeritage(tx, ty, day)` is pure + deterministic via a 32-bit
     hash (distinct multiplier from coop.pseudoRoll so streams don't
     correlate). Snapshot taken at `loadEgg` so the breed is locked in
     before the dawn hatch tick — reload-scumming is harmless.
   - PlacedCoop gains `heritage?: boolean[]` (per-slot flag).
     `addChicken(coop, heritage=false)` stores the flag at the new slot.
   - PlacedHatchery gains `incubatingHeritage` + `pendingHeritage` so
     the breed flag survives a full-coop pending pickup.
   - `hatcheryTick` HatchOutcome variants now carry `heritage: boolean`.
   - Hatch toasts + claim toasts + status line all name "heritage chick".
   - 13 new tests in `tests/hatchery-heritage.test.ts` including a
     200-day simulation showing heritage coops out-produce regular ones.

4. **pond-rotation (04b4a6e)** — `Pond species rotation`. When the
   pond is stocked-but-empty AND the player carries a different species,
   pressing `>` now swaps the pond to that new species.
   - New `reseedPond()` entry-point; PondOutcome gets a `restocked`
     variant carrying from/to + labels.
   - `interactPond` auto-tries reseed before falling through to the
     'nothing-pending' hint. Harmless when no swap is possible.
   - Status line now hints "...or press > with a different fish to swap"
     so the feature surfaces in the natural discovery loop.
   - `lastYieldDay` is preserved so the new species starts producing at
     the next dawn rollover — no special-case timing.
   - 9 new tests in `tests/fish-pond-rotate.test.ts`.

5. **spa-pass (e311998)** — `Bath house spa pass`. A new 700g punch
   card at Pip's cart carries SPA_PASS_PUNCHES=4 free soaks at the
   bath house.
   - `redeemSpaPass(player)` consumes one bag pass and credits 4
     punches into SpaPassState. takeBath auto-redeems an unredeemed
     bag pass before spending gold, then prefers punches over gold
     when any are available.
   - BathOutcome 'soaked' variant gains `paidWithPass` + `passesLeft`;
     bathFlavorLine names the punch path ("spa pass, 2 punches left").
   - SaveSnapshot.player.spaPass persists punches across reloads.
   - Cart catalog grew one row: Spa Pass between brass lantern + decor.
   - 11 new tests in `tests/spa-pass.test.ts`. Existing bath-house +
     bath-winter-discount inline fixtures updated for the new fields.

## Notes / call-outs

- No new keybinds this tick — every feature extended an existing flow
  (`>` for pond, `7` for compost/fertilizer apply, E for bath house,
  dawn ticks for storm + hatchery + compost). Digits 9 and 0 remain
  the only free digit slots. Letter keys remain exhausted.
- BathOutcome 'soaked' now carries FOUR optional fields beyond the
  basics: `pricePaid`, `discounted`, `paidWithPass`, `passesLeft`. Two
  test files had to be updated to include all four in their inline
  fixtures.
- The rare-fertilizer hash maps spring -> day 3. The existing compost
  test for "mints bags once the timer lapses" deposits on day 1 with
  finishOnDay=3, which is the spring rare day. The fix was to pass
  season=1 to `compostTick` so the bags land in the regular bucket as
  the test originally intended.
- Storm shelter pairing snapshots effective radii before mutating the
  array. Without that snapshot, removing the first half of a pair
  inside `consumeShelteringShelters` would retroactively shrink the
  partner's coverage to 1 mid-loop, causing crops 2 tiles from the
  surviving shelter to incorrectly count as un-sheltered.
- HERITAGE_HATCH_RATE = 0.18 produces ~17.8% over a 4200-sample sweep
  (asserted with a 3pp band so the rate is stable but not over-fit).
- Heritage breeding (passes propagate through new eggs without buying
  another spa pass) is now in the "next ideas" queue, plus a HUD hint
  for the rare fertilizer day, a stone-rim pond decor that lifts
  POND_MAX_PENDING, and a spa pass loyalty card.
