## What landed and why

### 1. premium-codex-section (3a394ea)
RecipeCodexRow widened with five premium-related fields
(hasPremium, premiumLine, premiumSellPrice, premiumCookedCount,
premiumOwned). New `premiumCookCounts` lazy player map +
`recordPremiumCook` / `premiumCookedCount` / `totalPremiumDishesCooked`
helpers — parallel to the existing `cookCounts` shape. The codex
panel now uses variable row heights (ROW_H=28 for plain rows,
ROW_H_PREMIUM=40 for unlocked egg recipes) and surfaces a pink
"x N ready / cooked x N" tally on the right edge + a dim pink
premium markup line ("Premium (breeder egg swap): Xg (+Yg)") under
the ingredient line. The header right-aligned tally grows with
"  -  N premium" only when totalPremiumDishesCooked > 0 so a fresh
save sees the same line it always did. Persistence carries
`premiumCookCounts` through serialize/apply.

### 2. lore-tab-footer (9e9b389)
New pure `loreTabFooter(player, category)` in lore.ts that
currently surfaces only on the Gems tab. Reads
`lifetimeHaulCount` + `lifetimeHaulGold` from mining-haul.ts and
locale-formats the gold ("career: 142 gems / 3,820g"). Drawn in
ROW_PIP_ON above the scroll indicator at y + h - 44. Tested against
the older-save mineHaul shape (no `lifetimeCounts` field) to
verify the lazy backfill keeps the helper crash-free.

### 3. double-batch-inn-toggle (d97114d)
`CookingMenu.confirm(player, mode='single' | 'double')` takes the
new mode arg (default 'single' for backwards-compat). Double mode
routes to `canCookDoubleBatch` / `cookDoubleBatch`; refuses
non-stamina-tea recipes with a new `not-eligible-double` outcome
branch. `CookOutcome` widened to carry `mode` + `yield` (1 single,
3 double). Game.ts wires `Shift+Enter` via `input.isPressed('shift')`
and bumps `recordCook` exactly `outcome.yield` times so achievement
milestones (e.g. "Pantry Cook" at 10) reflect the full bowl rather
than each cook invocation. Toast wording branches on mode
("Double-batched Berry Tonic! (+3)" vs "Cooked Berry Tonic!").
The cooking menu chrome surfaces the batch hint inline on the
selected row (replaces the "+Ng over raw" line for tea recipes)
and widens the footer hint to "Shift+Enter double-batch" when the
selected recipe is a tea.

### 4. compost-lifetime-ledger (85ccbad)
New `CompostLedgerState` (`lifetimeRecycledGold` +
`lifetimeBagsApplied`) bumped from inside `applyFertilizer` via the
new `recordApplied(player, recycledGold)` helper. Game-side, every
successful bag apply now grows two counters in parallel. New
`compost-master` achievement (catalog now has 17 badges, up from
16) lights up at `COMPOST_MASTER_MILESTONE_GOLD = 100`. The crop
journal panel surfaces `compostLedgerLine()` ("compost master:
128g recycled across 47 bags.") as a centered GREEN footer above
the close hint when the ledger is non-zero. Persistence carries
the new field via `SaveSnapshot.player.compostLedger`.

### 5. rumors-panel-filter (af25717)
Three-way filter cycle (`all` -> `bought` -> `skipped` -> `all`)
on the Rumors lore tab. Pure helpers in lore.ts: `RUMOR_FILTERS`
const tuple, `nextRumorFilter`, `applyRumorFilter` (filters rumor
rows by the entry's `count` field, which encodes `bought=1` /
`skipped=0`), and `rumorFilterLabel` for the chip. The
`LorePanel.rumorFilter` instance field resets to `all` on each
`open()`. `cycleRumorFilter()` is a no-op on tabs other than
Rumors so the keypress doesn't fight for shared keys. Game.ts
wires `f` as the cycle key inside the lore panel's input block;
the global fishing `f` handler now skips when the lore panel is
open so the keypress doesn't double-cast underneath the panel.
On the Rumors tab, the panel chrome surfaces a "filter: X (f to
cycle)" chip in ROW_PIP_ON at y + h - 44 and the close-hint
extends to mention `f cycle filter`.
