## What changed

### F1 — Rare-fertilizer HUD glance (0b5bec1)
- `compostStatusLine(bin, today, season?, dayOfSeason?)` now optionally surfaces:
  - empty bin: "RARE day in N days (day X)." pre-rare; nothing post-rare
  - staged batches: "one batch on the rare-day track." flag
  - ready bags: "(N RARE)" tag inline
- New helper `rareDayCountdownLine(season, dayOfSeason)` — pure formatter usable by any HUD
- game.ts:1380 — deposit toast now appends "RARE day — bags will be premium." when the batch
  lands on the season's rare day.
- 10 new tests in compost-rare-hud.test.ts. Two-arg legacy signature still works.

### F2 — Weather barometer (d8f6836)
- New `src/game/barometer.ts`: `BAROMETER_INVENTORY_KEY` ('barometer'), `BAROMETER_PRICE` (300g),
  `hasBarometer(player)`, `weatherDayAfterTomorrow(time)` with full +2 calendar carry.

  re-buy as 'already-owned'.

- `barometerBoughtLine()` runs on the cart purchase toast.
- 11 new tests cover hasBarometer, the +2 calendar carry at four wrap boundaries (day 6→1,
  day 7→2, season 3→0), the purchase happy path + re-buy guard + not-enough-gold rail,
  and the bought-line wording.

### F3 — Roving merchant rumor (e267935)
- New `src/game/cart-rumor.ts`: deterministic 32-bit hash mixes (season + 1) → a non-decor
  CART_CATALOG row. Decor filter prevents the same wallpaper from headlining two consecutive
  seasons. Pool sorted by key first so the hash bucket is stable against future catalog
  additions.
- `rumorToastLine(season)` rides the dawn toast on Pip's arrival day.
- `rumorFooterLine(season)` draws in the cart menu footer, above the "↑/↓ to choose" hint.
- cart-menu.draw() picked up an optional `time` arg; game.ts wires `this.time` through.
- 7 new tests cover the row pick, decor exclusion, determinism, rotation across seasons,
  label parity, and the toast + footer line shapes.

### F4 — Heritage breeding (20085fd)
- New constants in coop.ts: `BREEDER_EGG_INVENTORY_KEY` ('egg-breeder'),
  `HERITAGE_BREEDER_RATE` (0.4), `HERITAGE_BREEDER_MIN_HERITAGE` (2).
- PlacedCoop schema gained `breederEggs?: number` (optional, backward-compat).
- `coopTick(world, day)`: when a coop has ≥2 heritage chickens AND at least one fancy
  egg came from a heritage hen this morning, a deterministic `breederRoll(tx,ty,day) < 0.4`
  promotes one of those fancy eggs to a breeder. Per-dawn cap of 1.
- `collectEggs` + `collectEggsDetailed` route breeders into BREEDER_EGG_INVENTORY_KEY;
  the latter now returns `{ plain, fancy, breeder }`.
- `hatchery.loadEgg(hatchery, player, today)` PREFERS a breeder egg from the bag and forces
  `incubatingHeritage = true` bypassing the 18% HERITAGE_HATCH_RATE roll. Falls back to a
  regular fancy egg + the deterministic roll when no breeder is in the bag.
- game.ts toast updated: "Collected N eggs (incl. F fancy + B BREEDER)."
- fancy-eggs.test.ts toEqual updated for the new `breeder: 0` field.
- 12 new tests cover the roll determinism + uniform spread, the threshold gate (no breeder
  with <2 heritage), the fancy-day prerequisite, the per-dawn cap, the egg-count invariant
  (plain+fancy+breeder = chickens), the collect routing, and the hatchery breeder-vs-fancy
  preference + no-egg rail.

### F5 — Spa pass loyalty card (87fa150)
- BathState gained `totalSoaks?: number` + `soapsGifted?: number`.
- `takeBath()` increments totalSoaks on every soak (gold-paid AND spa-pass-paid). Uses

  of Perfumed Soaps — reload-safe.
- BathOutcome.soaked carries new `totalSoaks` + `soapsEarned` fields. THREE existing test
  files (bath-house, bath-winter-discount, spa-pass) had inline 'soaked' fixtures that
  needed updating with the new fields.
- `bathFlavorLine` appends "Bath house gifts you a Perfumed Soap for N lifetime soaks."
  on milestone soaks only.
- New `bathLoyaltyLine(player)` helper renders the running progress.
- Persistence: SaveSnapshot.player.bath now snapshots totalSoaks + soapsGifted; older v1
  saves restore cleanly (fields undefined → behaves like fresh defaults).
- 11 new tests cover the count bump (gold + spa pass), the every-10th gift, no-double-gift
  on reload-style state, the milestone vs non-milestone toast, and the fresh-player
  + mid-streak loyalty-line readouts.
